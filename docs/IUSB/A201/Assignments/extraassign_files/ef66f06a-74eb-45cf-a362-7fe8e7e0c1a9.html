This assignment is optional and is available for extra credit. It focuses on the use of Shoes in object-oriented fashion.

In the Shoes example subdirectory there is a file called simple-bounce.rb that implements an animated bouncing ball. A simplified version of this is listed below:

xspeed, yspeed = 8.4, 6.6
xdir, ydir = 1, 1

Shoes.app  :width=>300,:height=>300, :resizable=>false do
  background "#DFA".."#FDA"
  border black, :strokewidth => 6

  @icon = image "shoes.png", :left => 100, :top => 100

  x, y = self.width / 2, self.height / 2
  size = @icon.size
  animate(30) do
    x += xspeed * xdir
    y += yspeed * ydir
   
    xdir *= -1 if x > self.width - size[0] or x < 0
    ydir *= -1 if y > self.height - size[1] or y < 0

    @icon.move x.to_i, y.to_i
  end

end

The bouncing ball is really the icon for Shoes.

Now George took one look at this code and was really upset because it did not use classes and lumped all the code into one big code fragment. So he came up with the following class library:

class Ball
   attr_accessor :icon, :xdir, :x , :y 

  def initialize(x,y,image)
     @xdir = 2 + rand(10)
     @icon = image
     @max_x, @max_y = x,y
     @y = @icon.top = @max_y/2
     @x = @icon.left = @max_x/2
  end
 
  def incr
    @x += @xdir
    @xdir = -1 * @xdir.abs if (@x + @icon.width > @max_x)
    @xdir = @xdir.abs if (@x < 0)
    @icon.left = @x.to_i             
  end
 
  def faster
    @xdir *= 1.1
  end
 
end

He was so please with himself because the main program  reduced to the following code:

WIDTH = 300
HEIGHT = 300

Shoes.app :width=> WIDTH,:height=> HEIGHT, :resizable=>false do
   background "#DFA".."#FDA"
  border black, :strokewidth => 6
 
  @ball = Ball.new(self.width,self.height,(image "shoes.png"))

  button "faster" do
     @ball.faster
  end
 
  animate(30) do |i|
    @ball.incr
  end
end

At this point, George was running out of time and came by to ask for help. Here are the things that are needed. Credit will be given for the tasks you are able to complete. (The full amount of credit for completing all tasks equals 1.5 home assignments.).

Task 1: Vertical movement.

The class library worked well enough in the horizontal direction but was lacking control in the vertical dimension. You are asked to add to the class library so that it supports the vertical. The faster button should also speed up  the vertical movement by the same proportion.

Task 2: Slow button

Add a second button for slowing down the movement of the ball (Both in the vertical and in the horizontal directions).

Task 3: Quit button

Add a thrid button that exits the program. (Using the exit() method)

Task 4: Add a second ball

Add a second ball that bounces off the wall but passes throught the first ball. The two balls should  travel at different speeds. The faster and slower buttons should effect the speeds of both balls

Task 5: Add a class method Ball::collosion

Add a class method that will check if any two balls are close enough to collide and then adjusts the speed and direction of both to reflect the collosion. You do not have to implement the model for a fully elastic collosion. Instead, just implement a change of direction as if both balls had hit a wall at the moment of impact. This method should be called within the animation block to correct for the effects of a collosion.

Task 6: Add  a Third Ball

Increase the height and width of the application to 480 by 640 and add a third ball to the system. This ball should also respond to changes in speed and with collosions with any of the other two balls.
