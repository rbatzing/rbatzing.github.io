Instructions

In this assignment, I would like to ask you to submit a design for each of these games. Please submit the UML class inheritance and decomposition diagrams that describe the following games.

High Low

In this game, the computer selects a random number. The user will be asked to guess a number. If the guess is too high, the computer is to respond with the message "TOO HIGH" and if the guess is too low, the computer is to respond with the message "TOO LOW". In both cases, the user is invited to guess again and again until the guess is equal to the number. Once this happens, the computer is to respond with the message "YOU GUESSED IT" and display the number of tries the user took to find the correct number. After this, the computer is to invite the user to play another game. If the user responds with a "Y" the computer will guess another number and the game starts over again. Otherwise the computer will end the program.

 

Tic Tac Toe

In this game, the computer starts by drawing a game board consisting of a 3x3 matrix of squares. The user is 'X' and goes first to choose and mark a square in the matrix. The computer will then choose a suitable square and mark it with an 'O'. This goes back and forth with the user and the computer taking turns to choose squares on the game board. If either side is able to mark 3 squares in a line either down a column or across a row or down a diagonal, that sie wins the game. If all the squares are marked without, forming 3 of a kind in a straight line, the game is declared a tie. The computer will need to announce the outcome of the game and invite the user to play again. If the user chooses to play a new game, the computer resets the game board and a new game begins else the computer will end the program.
