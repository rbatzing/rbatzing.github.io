EXTENDING THE PETMINDER APPLICATION

Part 1. Writing YAML strings to a file.

Ruby provides support for a human-friendly, data rendering system called YAML (whose initials stands for YAML Ain't Markup Language). This library class allows the user to convert any Ruby object into a string that could be written to a file. The string is easy to read and could be edited in a text editor. At the same time, the use of punctuation and indentation within YAML allows YAML strings to be read back into memory.

For example, given this array of strings:

a = ['red','green','blue']

we can print the YAML equivalent:

puts a.to_yaml

giving this on the screen:

---
- red
- green
- blue

Write a method called store_yaml for the Collection class of your pet minder program that will write an object to a file called test.yml. Prove to yourself that you can use this method for storing Ruby objects by writing an array of  array, a hash, a strings and a pets to a file.  The following code should be used to
test your method:

mypets = Collection.new()

cat = Cat.new("Buffy")
myarray = [1,2,3]
myhash = {:smile => ':)', :frown => ':(', :crying => ';(', :joking => ';)'}
mytest = ['Hello',3.14159,dog,cat]
mypets.store_yaml(mytest)

Part 2. Reading a YAML string into an object.

The class method YAML::load can be used to read a yaml string into a Ruby object. Thus,

b = YAML::load( "--- \n- red\n- green\n- blue\n")

will create an array called b which is identical to the array in variable a in the section above.

Write a method called read_yaml for the Collection class of the pet minder that will read the objects from a file called test.yml. Prove to yourself that you can use this method for loading the assortment of Ruby objects you have stored in the file.
