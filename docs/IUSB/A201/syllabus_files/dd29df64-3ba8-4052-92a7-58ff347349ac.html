Material to be covered:

An introductory object-oriented programming course for non computer science majors, using the Ruby programming language. This course will start with a discussion of object oriented programming as a basis for modelling problem domains that would lead to the development of solutions using Ruby.

This course will cover key aspects of the Ruby programming language: Installing and Running Ruby, Object-oriented design,Building classes, Variable objects,Conditionals, Loops and iterators, Methods, Operators,Arrays and Hashes, Inheritance, Software testing, Profiling,Threads, Exceptions. The course will also explore key components of the Standard Class Library including File I/O,String methods, Regular expression methods, Shoe graphics, Interfacing with databases.

Rules of conduct:

While the use of chat rooms and wikis to address technical issues and build computing skills may a new experience for many in this class, it has become a common form of telecommunication especially among programmers of large Open Source software projects like Linux and Gimp. Even the business community has been using chat rooms and blogs to enhance communication between project participants living in different timezones. This form of communication is also relevant to online courses where the students are on different time schedules.

However, while these services can provide encouragement, insight and assistance, they are also governed by the Code of Conduct for Students. Submission of work done by someone else is considered plagurism and will deprive the offender of both the opportunity to master the skill and material covered and a passing grade for this course. Please do not post or transmit full solutions on these forums. Instead, it is recommented that if a section of code is being stubborn, a test file of no more than 4 lines of code be developed to demonstrate the problem. Test files developed this way can be freely shared and discussed in the wiki and chat rooms of this course without penalty.

To maximize the benefits of these services, the following guidelines are suggested:
  1. Be respectful and considerate: Disagreement is no excuse for poor behavior or personal attacks. Remember that a community where people feel uncomfortable is not a productive one. We will be covering a lot of ground and we will do better as a group.
  2. Be patient and generous: If someone asks for help it is because they need it. Do politely suggest specific documentation or more appropriate venues where appropriate, but avoid aggressive or vague responses. Be generous and fair in your use of these services. By helping others you are actually increasing your own understanding of the material.
  3. Assume people mean well: Remember that decisions are often a difficult choice between competing priorities. If you disagree, please do so politely. If something seems outrageous, check that you did not misinterpret it. Ask for clarification, but do not assume the worst.
  4. Try to be concise: Avoid repeating what has been said already. People will tend to ignore to long segments of code or background to a query. Making an answer longer makes it difficult to follow, and people often feel personally attacked if they receive multiple messages telling them the same thing.
